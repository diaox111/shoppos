# 🏪 商店智能收银 (ShopPOS)

> 面向中国小微便利店的 AI 智能收银系统 — Android APK 离线可用

**GitHub**: https://github.com/diaox111/shoppos

---

## 项目简介

ShopPOS 是一个**完全离线可用的智能收银 Android 应用**，专为中国街头巷尾的小微型便利店、烟酒店、社区超市设计。通过 AI 大模型实现商品条码/拍照自动识别、智能定价、区域化价格策略、保质期管理等功能，大幅降低小店数字化门槛。

## AI 赋能场景

| 功能 | AI 驱动方式 | 拟用模型 |
|------|------------|----------|
| 📷 **条码扫描** | ZXing 实时视频流解码 | — |
| 🤖 **商品识别** | 条形码 → AI 识别商品名称/分类/规格/建议售价 | **MiMo-V2.5** |
| 📸 **拍照识别** | 拍摄商品 → AI 读取包装信息自动填写 | **MiMo-V2.5** (multimodal) |
| 💰 **区域定价** | 根据店铺所在城市/商圈/景区自动调整售价系数 | MiMo 推理 |
| 🚬 **香烟双价** | 自动识别香烟类商品，分别给出单盒/整条价 | MiMo 推理 |
| 📅 **保质期推算** | 食品饮料类自动计算到期日期 | 本地计算 |

### 为什么选择 MiMo

1. **成本优势**: MiMo-V2.5 的 $0.40/M input tokens 定价适合高频扫码场景（便利店日均扫码 100-500 次）
2. **1M 上下文窗口**: 可一次性传入完整的商品分类体系 + 区域价格数据 + 历史销售记录
3. **OpenAI 兼容 API**: 与现有代码零改动对接，直接替换 `base_url` 和 `api_key`
4. **多模态能力**: MiMo-V2.5 原生支持图像理解，拍照识别无需额外模型

### 目标用户

- 中国 600 万+ 小微零售店主
- 日均扫码 50-500 次的社区便利店、烟酒店
- 需要简单易用、无需培训的收银工具

## 接入 MiMo API 的方式

当前项目使用 MiniMax API 进行商品识别，**可以无缝切换至 MiMo API**：

```typescript
// 仅需修改 3 行代码
const MIMO_URL = 'https://api.xiaomimimo.com/v1/chat/completions';
const MIMO_MODEL = 'mimo-v2.5'; // 或 mimo-v2.5-pro
const MIMO_KEY = process.env.NEXT_PUBLIC_MIMO_API_KEY;
```
